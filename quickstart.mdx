## Prerequisites

Before you begin, make sure you have **Node.js** installed on your machine. The project uses ES modules (`"type": "module"`) and targets modern Node.js runtimes.

<Steps>
  <Step title="Clone the repository">
    Clone the source repository to your local machine:

    ```bash
    git clone https://github.com/allocazione/allocazione.dev.git
    cd allocazione.dev
    ```
  </Step>

  <Step title="Install dependencies">
    Install all project dependencies. The project uses the following stack: Svelte 5, SvelteKit 2, Tailwind CSS 4, svelte-i18n, and anime.js.

    <CodeGroup>

    ```bash npm
    npm install
    ```

    ```bash pnpm
    pnpm install
    ```

    ```bash yarn
    yarn install
    ```

    </CodeGroup>
  </Step>

  <Step title="Sync SvelteKit types">
    Run the `prepare` script to generate SvelteKit's type definitions and sync the internal `.svelte-kit` directory. This is required before the dev server or build will work correctly.

    ```bash
    npm run prepare
    ```

    This runs `svelte-kit sync`, which scaffolds generated types based on your routes and config.
  </Step>

  <Step title="Start the development server">
    Launch the Vite dev server with hot module replacement:

    ```bash
    npm run dev
    ```

    The site will be available at `http://localhost:5173` by default. Vite loads the SvelteKit and Tailwind CSS plugins as defined in `vite.config.js`:

    ```js vite.config.js
    import tailwindcss from '@tailwindcss/vite';
    import { sveltekit } from '@sveltejs/kit/vite';
    import { defineConfig } from 'vite';

    export default defineConfig({
      plugins: [tailwindcss(), sveltekit()],
      build: {
        rollupOptions: {
          output: {
            manualChunks(id) {
              // Keep all svelte internals in one chunk to avoid circular dep warnings
              if (id.includes('node_modules/svelte/')) {
                return 'svelte-internals';
              }
            }
          }
        }
      }
    });
    ```
  </Step>

  <Step title="Build for production">
    Generate a fully static site using the `@sveltejs/adapter-static` adapter:

    ```bash
    npm run build
    ```

    The adapter is configured in `svelte.config.js`:

    ```js svelte.config.js
    import adapter from '@sveltejs/adapter-static';

    const config = {
      kit: {
        adapter: adapter({
          pages: 'build',
          assets: 'build',
          fallback: '404.html',
          precompress: false,
          strict: true
        }),
        prerender: {
          handleHttpError: ({ path, referrer, message }) => {
            // Suppress 404 error for missing favicon
            if (path === '/favicon.png') {
              return;
            }
            throw new Error(message);
          }
        }
      }
    };

    export default config;
    ```

    <Note>
      The static adapter outputs all pages, assets, and a `404.html` fallback into the `build/` directory. This folder is what you deploy to GitHub Pages, Cloudflare Pages, or any static host.
    </Note>
  </Step>

  <Step title="Preview the production build">
    After building, you can serve the `build/` output locally to verify the production result before deploying:

    ```bash
    npm run preview
    ```

    This starts a local Vite preview server pointed at the `build/` directory.
  </Step>
</Steps>

## Available scripts

All scripts are defined in `package.json`:

```json package.json
{
  "scripts": {
    "dev":     "vite dev",
    "build":   "vite build",
    "preview": "vite preview",
    "prepare": "svelte-kit sync || echo ''"
  }
}
```

| Script    | Command            | Description                                               |
| --------- | ------------------ | --------------------------------------------------------- |
| `dev`     | `vite dev`         | Start the Vite development server with HMR.               |
| `build`   | `vite build`       | Prerender all routes and output static files to `build/`. |
| `preview` | `vite preview`     | Locally serve the contents of `build/` after a build.     |
| `prepare` | `svelte-kit sync`  | Sync SvelteKit-generated types; runs automatically on install. |
