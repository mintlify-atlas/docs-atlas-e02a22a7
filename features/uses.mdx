The `/uses` page lists all of the hardware and software that Sel reaches for on a daily basis. It follows the community convention of [`/uses`](https://uses.tech) pages.

## Page header

The page heading is driven by the `uses.header` i18n key (`/uses.`). A `lastUpdated` timestamp from `siteConfig` is shown beside the heading when present:

```js config.js
lastUpdated: "2026-03-10"
```

```html uses/+page.svelte
{#if lastUpdated}
  <span class="text-[10px] text-gray-500 font-['Space_Mono'] tracking-widest lowercase">
    {$t('uses.last_updated', { values: { date: lastUpdated } })}
  </span>
{/if}
```

The i18n template for that label is `"last updated: {date}"`.

## Category data

Categories are read directly from the i18n JSON using `svelte-i18n`'s `$json()` function:

```js uses/+page.svelte
const categories = $derived($json("uses.categories") || []);
```

This means the category list is fully locale-aware — swapping locale automatically updates all category titles and item descriptions without any additional code.

Each category entry in the JSON has the shape:

```json en.json
{
  "title": "setup",
  "items": [
    { "name": "OS", "description": "Fedora Linux, Ubuntu Server, Windows 10", "url": "#" }
  ]
}
```

Items whose `url` is `"#"` are rendered as plain `<span>` elements. Items with a real URL are rendered as `<a>` tags that open in a new tab with a subtle arrow icon that appears on hover.

## Pending placeholder

After all category cards the grid always renders one additional dashed-border card as a placeholder for upcoming categories. Its label is sourced from `uses.pending`:

```html uses/+page.svelte
<div
  class="bg-[#0f0f0f]/40 border border-[#252525] border-dashed rounded-4xl p-8 flex items-center justify-center"
>
  <span
    class="text-gray-600 text-xs font-['Space_Mono'] tracking-widest lowercase animate-pulse"
  >{$t('uses.pending')}</span>
</div>
```

The English value for `uses.pending` is `"// connection pending..."`.
