## Email

The email address is read from `siteConfig.email`:

```js config.js
email: "me@allocazione.dev"
```

It is used in two places on the page:

1. The `mailto:` href on the primary CTA button:

```html contact/+page.svelte
<Button href={`mailto:${email}`}>
  {$t("contact.email_me")}
  <span class="ml-2 text-accent">&lt;/&gt;</span>
</Button>
```

2. A plain-text display below the button under the `direct_inquiry` label.

## Social links

The presence panel iterates over `siteConfig.socials`:

```js config.js
socials: [
  { name: "GitHub",   url: "https://github.com/allocazione" },
  { name: "Discord",  url: "https://discord.com/users/659022591071223819" },
  { name: "Telegram", url: "https://t.me/seleneftw" },
  { name: "Steam",    url: "https://steamcommunity.com/id/seloris" }
]
```

Each entry is rendered as an anchor that opens in a new tab with `rel="noopener noreferrer"`. On hover the row border transitions to `accent/40` and an arrow icon shifts right:

```html contact/+page.svelte
{#each siteConfig.socials as social}
  <a
    href={social.url}
    target="_blank"
    rel="noopener noreferrer"
    class="flex items-center justify-between p-4 rounded-2xl bg-white/2 border border-[#252525] hover:border-accent/40 hover:bg-white/5 transition-all duration-500 group/social"
  >
    <span class="text-sm text-gray-300 lowercase font-light group-hover/social:text-white transition-colors">
      {social.name}
    </span>
    ...
  </a>
{/each}
```

At the bottom of the presence panel a pulsing label reads `"// waiting for signal"` as a decorative footer element.
